# CTO Audit — trafico.live — 2026-06-10 (iteration 1)

## Headline

**Google has effectively not indexed the site.** Last 28 days (GSC): 6 clicks,
903 impressions, average position 44. Meanwhile every other crawler ranks us
fine — GA4 last 28d: Bing organic 80 sessions, ChatGPT 29, DuckDuckGo 11,
Yahoo 11, **Google organic 4**. This is a Google-specific discovery/crawl
failure, not a content-quality problem.

GA4 overall is growing from a tiny base: 189 sessions vs 108 prior 28d,
engagement rate 87% (up from 69%), bounce 13%. The product works; nobody from
Google search can find it.

## Root causes found (with evidence)

1. **`/trafico/{city}` pages were excluded from the sitemap** by a stale
   comment claiming they 301-redirect to `/ciudad/{city}`. The redirect was
   removed months ago (`next.config.ts` NOTE) and `/trafico/[city]/page.tsx`
   is the real page — our single best Bing entry point (19 sessions
   `/trafico/madrid` alone). Google was never told these exist.

2. **Google has `/trafico/madrid` and `/trafico/barcelona` recorded as 404**
   (URL Inspection API: "Not found (404)", lastCrawlTime 2026-04-14/16 —
   the era when the broken redirect chain existed). It never recrawled, so
   the 404 verdict stuck for ~2 months.

3. **Sitemap emitted 20 `/ciudad/{city}` URLs that have no route** —
   `src/app/ciudad/` has only `page.tsx`, no `[city]`. Live check:
   `/ciudad/madrid` returns 200 with the generic homepage title (soft-404 /
   duplicate content signal).

4. **Child sitemaps have never been downloaded by Google** — GSC sitemaps API
   shows `lastDownloaded: None` on every `/sitemap/N.xml` shard; the index
   itself last fetched 2026-05-24 with 0 URLs counted. News sitemap: 14
   submitted, 0 indexed, 1 warning.

5. **Crawl-budget poison at zero trust**: shard 0 ("core") had ballooned to
   9,280 URLs, of which thousands are ephemeral AIS vessel pages
   (`/maritimo/buques/{slug}` + `/recorrido` + `/historial/{year}`) that rot
   in days, plus a per-aircraft ICAO24 shard. Money pages were buried in
   churn on a domain Google already rations.

6. **Pages that ARE indexed rank** — `/provincias/48` pos 1,
   `/radares/CM-4000` pos 2, `/estadisticas/accidentes/badajoz` pos 5,
   `/espana/castilla-la-mancha/toledo` pos 2.7. Striking-distance list
   (pos 8–20, impr ≥200) is EMPTY — confirms the bottleneck is discovery,
   not on-page optimization.

## Fixes shipped this iteration (branch `fix/seo-sitemap-discovery`)

- `src/lib/sitemap-generator.ts`
  - Emit `/trafico/{city}` for the 10-city registry (priority 0.95, hourly).
  - Stop emitting `/ciudad/{city}` (route doesn't exist).
  - Stop emitting ephemeral vessel URLs (detail/recorrido/historial) from
    shard 0 — pages stay live and internally linked.
  - Drop the aircraft ICAO24 shard from the index (route still resolves).
- `next.config.ts`
  - 301 `/ciudad/{city}` → `/trafico/{city}` for the 10 supported slugs.
