00Las 8 decisiones cerradas

El roadmap está construido directamente desde 8 decisiones consecutivas del usuario. Cualquier ítem fuera de estas decisiones es Fase 2 post-launch y no se trabaja hasta después del lunes.

Q1Tipo de lanzamiento
A — Soft launch técnico. Consumer-focus. Sin B2B, API pública, MCP, Stripe tiers ni páginas profesional.
Q2Hero feature
B — 7 verticales con hubs pulidos. Resuelve el problema central: "las features se pierden".
Q3Profundidad hubs
A — Los 7 al mismo nivel full-feature. Hero map + stats + ticker + tiles + FAQ cada uno.
Q4Entity pages
A — Full 27K URLs SSG. Estaciones + líneas + aeropuertos + puertos + buques + carreteras + gasolineras + estaciones meteo/AQ.

Q7Live features nuevas

05Fase 2 — después del miércoles 22

Lo que se pospone expresamente (no se trabaja antes del launch):

  1. Stripe tier enforcement + billing portal — activar FREE/PRO/ENT en runtime con rate limits reales.
  2. API pública documentada — OpenAPI 3.1 + /api-docs renovado con Swagger UI.
  3. MCP server público — publicar @trafico/mcp-server en npm para que agentes IA consuman la data.
  4. /profesional landing B2B con pricing y casos de uso.
  5. Outreach B2B dirigido — 20-30 logística 3PL / aseguradoras / operadores con demo personalizada.
  6. OpenTripPlanner journey planning multimodal — el primer planner road+rail+ferry en España.
  7. Corridor intelligence dashboard — Madrid-Bcn, Madrid-Valencia, Madrid-Sevilla como producto B2B.
  8. Cross-joins analíticos — TrafficIncident × WeatherAlert, AccidentMicrodata × ClimateRecord, Vessel × PortCall dwell time.
  9. Submarine cables overlay (TeleGeography gratis), pipelines, power grid.
  10. 3D terrain + hillshade (SRTM/Copernicus DEM).
