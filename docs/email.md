# Email Configuration

trafico.live email is handled by **Cloudflare Email Routing** — no mail server needed.

## DNS Records

| Type | Name | Value | Priority |
|------|------|-------|----------|
| MX | trafico.live | route1.mx.cloudflare.net | 3 |
| MX | trafico.live | route2.mx.cloudflare.net | 79 |
| MX | trafico.live | route3.mx.cloudflare.net | 74 |
| TXT | trafico.live | `v=spf1 include:_spf.mx.cloudflare.net include:amazonses.com ~all` | — |
| TXT | cf2024-1._domainkey.trafico.live | `v=DKIM1; h=sha256; k=rsa; p=MIIB...` | — |
| TXT | _dmarc.trafico.live | `v=DMARC1; p=quarantine; rua=mailto:dmarc@trafico.live; pct=100` | — |

## Managing Rules

### Via Cloudflare API

```bash
# List rules
curl -s "https://api.cloudflare.com/client/v4/zones/ZONE_ID/email/routing/rules" \
  -H "X-Auth-Email: $CF_API_EMAIL" -H "X-Auth-Key: $CF_API_KEY"

# Update catch-all
curl -s -X PUT "https://api.cloudflare.com/client/v4/zones/ZONE_ID/email/routing/rules/catch_all" \
  -H "X-Auth-Email: $CF_API_EMAIL" -H "X-Auth-Key: $CF_API_KEY" \
  -H "Content-Type: application/json" \
  --data '{"matchers":[{"type":"all"}],"actions":[{"type":"forward","value":["new@email.com"]}],"enabled":true}'
```

### Via Cloudflare Dashboard

Dashboard → trafico.live → Email → Email Routing → Routing Rules

## Notes

- **amazonses.com** is in the SPF record from legacy SES setup — can be removed if SES is no longer used for transactional email
- DKIM key was generated by Cloudflare (`cf2024-1._domainkey`) — not a custom key
- DMARC policy is `quarantine` — non-authenticated mail goes to spam, not rejected
- Destination `mj@blue-mountain.es` was verified automatically (same Cloudflare account)
